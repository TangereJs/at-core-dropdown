paper-menu demo

  • provides styling for itself
  • provides iron-selected css class for items that are tapped/clicked on
  • provides focus handling for itself; user can TAB into paper-menu
  • once paper-menu has keyboard focus, user can cycle through items with ARROW keys
  • selecting "items" with enter or space doesn't work; paper-item does this
  • closing the dropdown with ESC is very difficult; paper-menu needs focus to handle ARROW keys but also stops bubbling of keyboard events, so core-dropdown can not close itself with ECS because paper-menu intercepts ECS key
  • the same holds true for ENTER key where we want to select the item and close the dropdown in one step

Dropdown position is bottom left
